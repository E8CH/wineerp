아래 기능을 포함한 크로스 안드로이드 모바일 서비스의 핵심 기능 구현을 하려고해.
최소 기능만 구현하려고해.




- 기능 범위
  - 직원 : 가입 > 로그인 > 카메라 > 바코드 스캔 > 와인 라벨링 사진촬영 > DB에 모델이 있으면 모델명+연식 띄우기(화면에는 사진도 나와야 함) > DB에 모델이 없으면 새로 추가하시겠습니까? 하고 라벨링에서 불러온 기본 모델명 적어주고 수정 가능하게 할 것 > 수량지정 > 입고완료 버튼 > 등록 이후 수량 수정 버튼도 있어야 함 > 일간/주간/월간 데이터를 게시판 형태로 볼 수 있어야 함 > 종합 레포트가 있어야 하며, 입고항목에 대한 그래프형태로 분석할 수 있어야 함 > 엑셀로 다운로드 받을 수 있어야 함
  - 관리자: 가입 > 로그인 > 일간/주간/월간 데이터를 게시판 형태로 볼 수 있어야 함

- 시연에 사용하는 기술 스택
  - 1. 관리자용 반응형 웹 (Next.js)
  - 2. 사용자(고객/고수)용 모바일 앱 (flutter)
  - 3. 통합 API (FastAPI)
  - 4. DB (PostgreSQL)
  - 5. 배포 (Railway : 프론트엔드, 백엔드, PostgreSQL)

-DB 설계
직원,관리자 : e-mail(PK), password
게시글 : increase number (PK), e-mail(회원:직원), 사진, 글, 게시날짜(yyyyMMhhss.zzz), 상태(예약중,판매중,거래완료)

-조건
와인의 거래 과정에서 입고/재고관리를 하는 서비스가 필요해
와인이 들어올때에 하루에 1-100개가 들어올 때가 있음
거래명세서의 항목과 실제 와인의 항목이 다른 경우가 있음. 따라서 거래명세서는 설득력이 떨어짐
같은 모델이라 할지라도 라벨링이 미묘하게 달라서 연식이 다른경우가 있거나 같은 바코드라도 상세 모델이 다를지는 알 수 없는 상태
고객은 이것을 이용해서 입고시 카메라를 찍어서 바코드나 사진을 찍어서 모델명을 분류하고 이에 따른 수량을 입력시키고 싶어 함
다만 데이터는 모바일 위주로 하면 안되고, 서버가 있어야 함
입고에 대한 재고관리에 대해 일간,주간,월간 데이터가 나와야 함


- 참고 사항
와인 관련된 서비스나 분위기의 UX/UI를 서치하여 톤앤 매너를 잡아야 하며, 다만 한국인의 정서에 맞아야 한다.
