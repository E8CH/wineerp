요약 Executive Summary

부정확한 거래명세서 대신, 스마트폰으로 바코드와 라벨을 찍어 품목을 확정하는 모바일 우선 입고·재고관리 서비스를 제안합니다.

매일 들어오는 와인(하루 1~100병, 연 약 1,000종)을 종이 명세서와 대조해 손으로 입력하는 현재 방식은 시간이 많이 들고 오입력이 반복됩니다. wineerp는 현장 담당자가 병을 찍으면 사전 구축된 DB와 매칭해 모델명·빈티지·사진을 자동으로 띄우고, 수량만 입력하면 입고가 끝나는 흐름을 제공합니다. 전체 개발에 앞서 1주 규모 POC로 핵심 흐름과 최대 미지수(바코드로 빈티지를 구분할 수 있는가)를 실물로 먼저 증명합니다.

1 사업 배경 및 문제

  • 명세서를 신뢰하기 어렵다. 와인명이 복잡하고 수입사마다 표기가 달라, 종이 명세서와 실제 입고 품목이 어긋납니다.
  • 검수가 곧 병목이다. 사람이 하나하나 대조·입력해 시간이 과도하게 들고, 하루 100병이 몰리면 부담이 급증합니다.
  • 와인은 식별이 까다롭다. 같은 이름도 빈티지가 다르면 별도 품목이며, 연식은 실물 라벨을 봐야 확정됩니다.
  • 핵심은 품질이 아니라 "정확한 품목 + 수량". 고객이 원하는 검수는 품질 검사가 아니라 무엇이 몇 개 들어왔는지입니다.

2 제안 개요

"찍으면 뜬다" — 네 단계로 끝나는 입고.

  1. 스캔 + 라벨 촬영 — 바코드로 1차 매칭, 라벨 사진으로 빈티지 등 바코드가 못 나누는 차이를 보정.
  2. 자동 식별 — DB에 있으면 모델명·빈티지·사진 표시. 없으면 라벨을 AI로 분석해 모델명을 채워 주고 담당자가 확인·수정.
  3. 수량 입력 → 입고 완료 — −/숫자/+ 로 한 손 입력, 명시적 완료. 완료 후 내역에서 수정 가능.
  4. 집계·리포트 — 일/주/월 조회, 그래프 종합 리포트, 엑셀 내보내기(회장 보고용).

데이터는 로컬이 아니라 서버 중심으로 축적되어, 쌓일수록 식별이 빠르고 정확해지는 고객사 고유 자산이 됩니다. 발표용 화면 목업을 함께 제출합니다.

4 사용자 경험 · 화면

  • 모바일 우선 · 안드로이드 폰: 한 손 세로 조작, 하단 고정 "완료" 버튼, 현장 조명에서도 잘 보이는 큰 글씨.
  • 톤앤매너: 딥 네이비 + 화이트, 깔끔·고시인성. 병 사진이 주인공.
  • 갤럭시 폴드 대응: 커버(좁고 긴 화면)·메인(정사각 화면) 양쪽 해상도와 접기/펴기 전환까지 고려.
  • 상세 사양은 UX 디자인 사양서와 화면 목업으로 별도 제출합니다.

6 추진 범위 — POC vs 본 제품

1차 · POC (1주)

지정 와인 10종 기준. 스캔·촬영 → 식별 → 신규등록 → 수량 → 입고·수정의 핵심 흐름을 end-to-end로 구현하고, 바코드↔빈티지 판정을 도출합니다.

2차 · 본 제품 (POC 검증 후)

전 품목(~1,000종) DB, 일/주/월 리포트·그래프·엑셀, 관리자 집계 화면(모바일 우선), 필요 시 웹 병행.

범위 안내: 미팅 합의(1주·3페이지·라벨 10종·약 200만 원)는 POC의 하한 기준입니다. 다만 확정된 기능(관리자 역할 + AI 라벨 추론 + 리포트 전체 + 초기 세팅)은 이 조건을 넘어설 수 있어, POC 계약 범위와 초과분 처리를 착수 전 함께 확정하고자 합니다.
