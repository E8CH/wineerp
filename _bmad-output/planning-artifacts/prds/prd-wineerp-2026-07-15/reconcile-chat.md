## 2. 누락·왜곡 (GAP) — 핵심

### GAP-1. POC 상업조건 중 "포트폴리오↔할인" 레버가 통째로 누락
- **근거(L77):** 화자 A가 명시적으로 제안 — "저희도 이것들을 포트폴리오로 쓸 수 있게 되면… 회사 로고가 될 수도 있고 어떤 사례를 들 수도 있는데… 이 정도 범위를 오케이 하시면 저희가 일부 금액을 퍼센테이지로 좀 낮춰 드릴 수 있는". 즉 **포트폴리오 활용 동의 → 견적 할인**이라는 상업 협상 조건이 제시됨.
- **PRD 상태:** §6·§8 OQ-1은 "1주·3페이지·200만원"만 담고, **포트폴리오 노출 범위 동의 여부 / 그에 연동된 할인**은 어디에도 없음. 고객이 "회장님께 결제 올려보고"(L95) 판단하는 상업 변수인데 산출물에서 실종.
- **영향:** POC 계약 금액 협상의 핵심 지렛대가 빠져 회장 보고 시 조건 누락 위험. OQ-1에 상업조건 항목으로 추가 필요.

### GAP-2. LLM/OCR 라벨 모델명 추론(FR-6)은 고객 미요청 — 주입된 범위 확대
- **근거:** 미팅 전체에서 고객이 요구한 흐름은 "**바코드를 DB와 매칭 → 바코드 찍고 수량만 입력 → 자동 입고**"(L15, L27). 고객은 이미 **내부적으로 상품명·코드 시스템을 보유**한다고 진술("상품명이라든가 코드 같은 것들은 저희가 내부적으로 시스템을 하긴 하는데" L27). 라벨을 AI/LLM/OCR로 읽어 모델명을 자동 채운다는 요구는 **고객이 한 적 없음**. 화자 A도 "이 물건에 대한 판단을 어떻게 할 건지 정보를 주고받아 확인"(L61, L65) 수준으로만 언급, 매칭 테이블 제공 가능 여부를 물음.
- **PRD 상태:** FR-6, UJ-2, §3 Glossary, addendum이 LLM(Gemini/OpenAI) 라벨 추론을 MVP In-Scope로 확정.
- **영향:** ① 근거 없는 기능 주입. ② PRD 스스로 §4 Notes·OQ-1에서 "OCR 포함 시 200만원·1주 초과"라고 인정 → 고객이 합의한 상업조건을 자기 손으로 깨는 구조. ③ 고객은 내부 코드 체계가 있으므로 신규등록 자동추론의 실제 필요/우선순위 미확인. **최소한 POC에서 LLM은 옵션/후속으로 내려야 정합.** (PRD가 NOTE·OQ로 자각은 하고 있으나 In-Scope로 못박은 것이 모순.)

### GAP-3. "검수 시간 절반(50%)" 목표는 고객이 정량화한 적 없음 — 임의 수치
- **근거(L27):** 고객은 "검수하는 데 걸리는 시간이 확 줄어들 것 같다"고만 정성 표현. 50%라는 수치 언급 없음.
- **PRD 상태:** §1 Vision "검수 시간을 절반으로 줄이고", SM-1 "검수·입력 시간 50% 단축"을 Primary 지표로 확정.
- **영향:** 검증 불가능한 임의 KPI가 성공 기준으로 고정됨. `[ASSUMPTION]` 태그 없이 단정 → 근거 표기 필요.

### GAP-4. 플랫폼 "안드로이드 우선"은 고객 미언급 — 임의 가정
- **근거:** 고객은 "모바일 환경에서 돌아갈 수 있게"(L71)만 언급, OS 지정 없음.
- **PRD 상태:** §11 "직원 앱: 모바일(안드로이드 우선)".
- **영향:** 경미하나 근거 없는 단정. UX/아키텍처 전 확인 필요(iOS 사용 여부 미확인).

## 3. 부수 관찰 (경미)

- **3.1 바코드 vs 사진 — 방법론 미결이 PRD에선 둘 다 필수화.** 미팅에서 화자 A는 "바코드 찍는 방법도 있고 사진 찍는 방법도 있다… 이 물건 판단을 어떻게 할지만 정하면 된다"(L57–65)며 **택일/미결** 상태로 남김. PRD는 FR-3(바코드)+FR-4(라벨사진)를 매 흐름에 **동시 필수**로 넣음. 라벨 촬영이 항상 필요한지는 미검증 → OQ-3에 흡수는 되나 명시적 미결 표기 권장.
- **3.2 3페이지 = "돌아가는 시퀀스 3페이지"의 의미.** 화자 A "일주일 내 세 페이지 정도로 기본 시퀀스와 10개 레이블 기능 확인"(L89). PRD OQ-1이 "3페이지"를 언급하나, 관리자 역할+리포트 전체+OCR을 In-Scope로 넣으면 3페이지로 불가 — PRD도 §4 Notes에서 초과 인정. GAP-2와 동일 뿌리.
- **3.3 첫 외주·회장 지시·빠른 가시화 맥락(L75, L83, L85)** — "빠른 결과물"을 위한 POC 분리 논리는 §6·§8에 잘 반영됨. OK.
- **3.4 수량은 수기 입력.** 고객/화자 A 모두 "수량은 수기"(L15, L65) → FR-7에 반영. OK.
