# Story 1.5: M3 네이비 앱 셸 & 접근성 기반 (UX-DR1·2·3·14·15)

Status: review

## Story

As a 사용자,
I want 일관된 테마·내비게이션·접근성 기반의 앱 셸을,
so that 모든 화면이 시인성 높고 한 손으로 조작 가능하다.

## Acceptance Criteria

1. **테마**: `ColorScheme.fromSeed(딥 네이비 #123E7C)` 기반 M3 라이트 테마. Primary #123E7C, Primary Strong #1766B0, 배경 #F6F7F9, Surface #FFFFFF, 상태색(Success #2E7D53/Warning #B8860B/Error #D32F2F), 카테고리(네이비·마룬 #9B1B1B·골드 #B8860B). 시인성 우선 큰 타이포 스케일.
2. **내비게이션**: 하단 4탭(스캔·내역·리포트·재고), 홈=스캔. go_router 셸 라우트.
3. **접근성**: 인터랙티브 요소 최소 48dp, 네비 목적지에 semantic 라벨, 색+아이콘 병기(라벨 텍스트 상시).
4. **반응형/폴드**: 단일 컬럼, 가로스크롤 없음. 최소폭 280dp에서 레이아웃 무결(Compact).
5. **앱 부팅**: `ProviderScope`(Riverpod) + `MaterialApp.router`(테마+라우터). `flutter analyze` clean.

## Dev Notes

- **디자인 시스템**: Flutter Material 3(useMaterial3 기본). ColorScheme.fromSeed 후 필요한 색만 override. [Source: ux-design-specification.md#Color System / Design System Foundation]
- **폰트**: Pretendard/Noto Sans KR는 자산 번들 필요 → 이번엔 시스템 폰트(안드로이드 Noto Sans KR 기본)로 두고 큰 타이포 스케일만 적용. 폰트 번들은 후속(자산 추가) 시.
- **탭 구성**: 스캔(홈)·내역·리포트·재고. 리포트는 manager 전용이나 셸에는 노출, 접근 제어는 화면/데이터 단(Epic 5). [Source: ux-design-specification.md#Navigation Patterns]
- **상태관리**: Riverpod `ProviderScope`로 앱 감싸기(실제 프로바이더는 기능 스토리에서). go_router는 `routerConfig`로 주입.
- **접근성**: NavigationBar 기본이 48dp+ 충족. 목적지 label 상시 표시(색만 의존 금지). [Source: ux-design-specification.md#Accessibility Strategy]
- **반응형**: 셸은 단일 컬럼. 화면 본문은 `SafeArea`+세로 스크롤. 최소폭 280dp 고려(가로 고정폭 금지).
- 기존 골격(Story 1.1)의 `lib/core/env.dart`, `features/` 디렉터리 재사용. 기본 `main.dart`(counter) 대체.
