# Story 1.3: 직원 가입/로그인 (FR1)

Status: review

## Story

As a 현장 검수 직원,
I want 이메일+비밀번호로 가입하고 로그인하기를,
so that 스캔·입고 기능에 접근할 수 있다.

## Acceptance Criteria

1. **가입**: `POST /api/v1/auth/signup`(email, password) → role=staff 계정 생성, 비밀번호는 해시 저장. 응답은 UserRead(id·email·role), 평문/해시 미노출.
2. **중복 거부**: 동일 email 재가입 시 400.
3. **로그인**: `POST /api/v1/auth/login`(OAuth2 password form) → 유효 시 JWT access_token 발급(`{access_token, token_type: "bearer"}`), 무효 시 401.
4. **현재 사용자**: `GET /api/v1/auth/me` → 유효 베어러 토큰이면 현재 UserRead 반환, 없거나 무효면 401.
5. **인증 가드**: 보호된 리소스는 토큰 없으면 401(미인증 → 로그인 유도의 서버측 근거).

## Dev Notes

- **스택**: pyjwt(JWT), fastapi OAuth2PasswordBearer/OAuth2PasswordRequestForm, python-multipart(폼). pyproject 포함.
- **역할 기본 staff**: signup은 항상 staff 생성(관리자 승격/생성은 Story 1.4). [Source: epics.md#Story 1.3]
- **crud/security 재사용**: `crud.user.create_user`·`get_user_by_email`, `security.verify_password`·`get_password_hash` 이미 존재(Story 1.2) — **재구현 금지**.
- **와이어 규약**: JSON snake_case, 경로 `/api/v1/auth/*`, 에러는 FastAPI `{"detail": ...}`(401/400). [Source: architecture.md#Implementation Patterns]
- **JWT**: sub=user id(str), exp=now+ACCESS_TOKEN_EXPIRE_MINUTES, alg HS256, key=settings.SECRET_KEY. 토큰 무효/만료→401.
- **DB 의존성**: `get_db_session`은 `core/db.get_session` 사용. 테스트는 `app.dependency_overrides`로 SQLite 세션 주입.
- **보안**: 로그인 실패는 email/password 구분 없이 동일 401(사용자 열거 방지).
