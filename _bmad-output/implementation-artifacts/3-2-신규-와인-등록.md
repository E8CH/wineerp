# Story 3.2: 신규 와인 등록 (FR6, UX-DR8)

Status: review

## Story

As a 직원,
I want 미매칭 와인을 라벨 한 장으로 즉석 등록하기를,
so that 긴 와인명을 타이핑하지 않고 새 마스터를 만든다.

## Acceptance Criteria

1. **등록 엔드포인트**: `POST /api/v1/wines`(인증) — `{producer, model_name, vintage?, region?, country?, grape?, barcode?, representative_image_key?}` → 201 `{product_id, vintage_id}`.
   - `vintage`는 **nullable**. NV는 인식 실패가 아니라 1급 상태다(AR2).
   - `barcode`가 있으면 `get_or_create_barcode` + `link_barcode_to_product`로 연결(N:M, AR3).
   - 기존 `crud.wine` 함수 재사용 — 재구현 금지.
2. **라벨 사진 필수**: 등록 진입 시 라벨 촬영이 선행된다. 사진 없이는 [모델검색]도 등록도 불가.
   - 리서치상 매칭 실패의 큰 축이 **바코드 없는 와인(~50%)**이고, 사진은 기록·재매칭·추론의 공용 소스다(FR4).
3. **AiInferenceField(UX-DR8)**: [모델검색]을 누르면 `POST /inference/label` 결과가 모델명 필드에 채워지고, **파란 "AI 추론" 태그**가 붙는다.
   - 자동 채운 값은 **항상 수정 가능**하며 태그가 그 사실을 드러낸다(SM-C2 맹신 방지).
   - 저신뢰면 골드 경고 배지 + "확인 필요" 문구를 병기한다(색 단독 전달 금지).
4. **[직접입력] 폴백 상시 노출**: 추론 대기 중에도 [직접입력]이 보이고 눌린다. 차단 모달 금지(UX-DR13).
   - 실패·저신뢰·사용자 선택 어느 쪽이든 **즉시** 수동 입력으로 전환된다.
5. **빈티지 입력에 NV를 1급으로**: 연도 입력 필드 + **[NV] 토글**. NV 선택 시 연도 입력이 비활성되고 `vintage=null`로 전송된다.
   - ⚠️ 연도를 비워두는 것으로 NV를 표현하지 말 것 — "아직 입력 안 함"과 "NV로 확정"이 구분되지 않는다.
6. **등록 직후 입고로 연결**: 등록 성공 시 생성된 `vintage_id`로 곧바로 수량 입력·[완료] 흐름(`ReceivingPanel`)에 이어진다.
7. **검증**: 백엔드 — 201·바코드 링크·NV 등록·중복 바코드 재사용·필수값 422·미인증 401. Flutter — 사진 없으면 등록 불가 · 추론 결과가 태그와 함께 채워짐 · 대기 중 [직접입력] 활성 · 저신뢰 경고 · NV 토글이 null 전송 · 등록 후 입고 패널 연결.
8. **변이 검증**: "사진 필수" 가드, "AI 추론 태그", "NV 토글의 null 전송"을 각각 제거하면 테스트가 실패해야 한다. ([[test-mutation-verification]])

## Dev Notes

### 의존성 결정: `camera`가 아니라 `image_picker`

**하드웨어 경합이 이유다.** `mobile_scanner`가 이미 카메라를 점유하고 있어서 `camera` 패키지를 더하면 두 플러그인이 같은 장치를 두고 다툰다(스캔 화면 위에 등록 패널이 뜨는 구조라 동시 활성 가능). `image_picker`는 시스템 카메라 앱에 넘겼다가 결과만 돌려받으므로 충돌하지 않는다.

부차적으로, 라벨 촬영은 **병마다가 아니라 새 와인마다 한 번** 일어나는 예외 경로다. 인앱 프리뷰의 UX 이득이 경합 리스크와 코드량을 정당화하지 못한다. 되돌릴 수 있는 결정이며, 인앱 촬영이 필요해지면 `mobile_scanner`의 카메라를 공유하는 쪽을 먼저 검토할 것.

### AI 값은 초안이지 답이 아니다 (SM-C2)

PRD의 카운터 지표 SM-C2는 **"LLM 추론 맹신 방지"**다. 자동 채움 편의 때문에 직원이 확인을 건너뛰어 오등록이 늘면 이 기능은 순손실이다. 따라서:
- 자동 채운 값에는 **항상 태그가 붙는다**(사용자가 출처를 안다)
- 필드는 **항상 편집 가능**하다(읽기 전용으로 잠그지 않는다)
- 저신뢰는 **아이콘+문구**로도 전달한다(색 단독 금지)
- 추론 결과를 **자동 저장하지 않는다** — 아키텍처 안티패턴 목록의 "LLM 결과 무확인 자동 저장"

### NV를 빈 값으로 표현하지 말 것

`vintage=null`은 두 가지를 의미할 수 있다: "NV로 확정" vs "아직 입력 안 함". 전자는 유효한 재고 단위이고 후자는 미완성 입력이다. 명시적 [NV] 토글을 두면 이 둘이 구분되고, 직원이 실수로 빈 채 넘기는 것도 막힌다. 샴페인 85~95%·셰리 ~98%가 NV라 드문 경우가 아니다.

### 폴백은 실패 후가 아니라 처음부터 보인다

UX 사양: *"대기: 인라인 진행 표시 + 폴백 링크 상시, 전체 차단 모달 금지"*. 추론이 3~5초 걸리는데(NFR5) 그동안 직원이 아무것도 못 하면 현장 리듬이 끊긴다. [직접입력]은 추론 중에도 눌린다.
