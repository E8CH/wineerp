# Story 1.1: 프로젝트 부트스트랩 & 배포 파이프라인

Status: ready-for-dev

## Story

As a 개발팀,
I want 백엔드·프론트 스타터를 초기화하고 Railway 배포·Dart 클라이언트 생성 파이프라인을 세우기를,
so that 이후 모든 기능을 일관된 기반 위에서 구현하고 즉시 배포할 수 있다.

## Acceptance Criteria

1. **모노레포 구조 & 로컬 기동**: `full-stack-fastapi-template`의 backend를 도입하되 `/frontend`(React)를 제거하고, `flutter create`로 `app/`을 생성한다. `backend/`(FastAPI+SQLModel+Alembic+Docker)와 `app/`(Flutter M3)로 구성된 모노레포에서 로컬 `docker-compose up`으로 backend+PostgreSQL이 기동된다.
2. **헬스체크 & 배포**: `GET /api/v1/health`(또는 `/api/v1/utils/health-check`)가 200을 반환하고, Railway에 backend + PostgreSQL이 배포되어 원격 헬스체크가 200을 반환한다.
3. **Dart 클라이언트 생성**: FastAPI가 노출하는 OpenAPI 스키마(`/api/v1/openapi.json`)로부터 Flutter용 Dart 클라이언트를 생성하는 스크립트가 동작하고, 생성물이 `app/lib/data/`에 놓인다.
4. **시크릿 위생**: LLM(Gemini/OpenAI)·스토리지(R2) 시크릿은 `.env`(유료 티어 전제)로 주입되고, `.env`가 `.gitignore`에 포함되어 커밋되지 않는다. `.env.example`에 키 이름만 문서화한다.

## Dev Notes

### 아키텍처 패턴·제약 (반드시 준수)
- **모노레포 2계층**: `backend/`(FastAPI+SQLModel+PostgreSQL+Alembic) + `app/`(Flutter). 관리자 웹(Next.js)은 범위 밖 — 만들지 말 것. [Source: architecture.md#Project Structure & Boundaries]
- **스타터는 공식 템플릿**: `fastapi/full-stack-fastapi-template`(MIT). 인증(JWT·이메일/비번)·User 모델·Alembic·Docker·CI를 제공하므로 **재발명 금지**. React 프론트만 제거. [Source: architecture.md#Starter Template Evaluation]
- **Ports & Adapters 골격**: `app/services/`(Port 인터페이스), `app/adapters/`(벤더 구현)를 이번에 **디렉터리로 스캐폴드**만. 실제 어댑터(Gemini/OpenAI/LWIN/R2)는 Story 2.1·2.3·3.1. 라우트가 벤더를 직접 호출하지 않도록 경계만 세운다. [Source: architecture.md#API & Communication Patterns]
- **와이어 규약**: API JSON=snake_case(Pydantic 기본), 경로 `/api/v1/{복수명사}`, PK=UUID. Dart 모델은 camelCase + `@JsonKey`/생성 클라이언트가 흡수. [Source: architecture.md#Implementation Patterns]
- **배포**: Railway(backend Docker + PostgreSQL). 이미지 스토리지=Cloudflare R2(후속 스토리에서 실사용). [Source: architecture.md#Infrastructure & Deployment]
- **시크릿·유료 티어**: LLM 키는 **Billing 활성화된 유료 계정**이어야 함(무료 Gemini는 고객 데이터 학습·사람 검토 — 계약 위반은 아니나 고객 설명 불가). `.env`는 절대 커밋 금지. [Source: research 2026-07-17 §Gemini 약관 / architecture.md AR9]
