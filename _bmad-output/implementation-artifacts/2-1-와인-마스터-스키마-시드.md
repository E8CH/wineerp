# Story 2.1: 와인 마스터 스키마 & 시드 (AR2·AR3·AR8)

Status: review

## Story

As a 개발팀,
I want 2계층 와인 식별 스키마와 LWIN·시연용 시드를,
so that 스캔 매칭이 실제 데이터 위에서 동작한다.

## Acceptance Criteria

1. **2계층 스키마**: `wine_products`(producer·model_name·region·country·grape·lwin7 nullable) 1:N `wine_vintages`(wine_product_id FK·vintage **nullable**·lwin11·representative_image_key). `vintage`는 NULL 허용(NV=1급).
2. **바코드 N:M**: `barcodes`(code unique) + `barcode_wine_product_link`(N:M) — 하나의 바코드가 여러 WineProduct에, 하나의 WineProduct가 여러 바코드에 연결 가능.
3. **마이그레이션**: Alembic로 4개 테이블 생성.
4. **바코드에 빈티지 미인코딩**: 바코드 문자열에서 연도 파싱 로직 없음.
5. **시드**: LWIN CSV 로더(있으면 로딩) + 시연용 와인 마스터 10종(바코드 포함) 시드. lwin7/lwin11을 내부 표준키로 보관.
6. **바코드 조회 helper**: 바코드 code → 연결된 WineProduct 목록 → 각 WineVintage 조회(2.4 매칭 기반).

## Dev Notes

- **핵심 근거**: 리서치 2026-07-17 — 로트≠빈티지, 바코드는 producer+wine+size까지만, 바코드↔와인 N:M. `vintage` nullable(NV 다수). [Source: architecture.md#Data Architecture / research]
- **명명**: 테이블 snake_case 복수, FK `{entity}_id`, PK UUID. [Source: architecture.md#Implementation Patterns]
- **관계 처리**: ORM relationship 대신 FK + 명시 쿼리(단순성). 매칭(2.4)은 code→barcode→link→products→vintages 경로.
- **LWIN**: 실제 CSV는 폼 뒤(미보유) → 로더는 `data/lwin/*.csv` 존재 시 로딩, 없으면 스킵. 시연 10종은 lwin7/lwin11 예시값 포함. [Source: architecture.md AR8]
- **재사용**: `core/db.get_engine`, 기존 Alembic 베이스라인(0001). 이 마이그레이션은 down_revision=0001_users.
- **테스트 DB**: SQLite in-memory(psycopg 불필요). vintage nullable·N:M 양방향 검증.
