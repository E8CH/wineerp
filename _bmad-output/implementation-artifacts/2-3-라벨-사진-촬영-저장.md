# Story 2.3: 라벨 사진 촬영·저장 (FR4, AR4)

Status: review

## Story

As a 직원,
I want 병 라벨 사진을 촬영·저장하기를,
so that 기록·매칭 보정·LLM 추론의 공용 소스로 쓸 수 있다.

## Acceptance Criteria

1. **StoragePort 추상화**: 오브젝트 스토리지 Port + 어댑터(로컬/R2). env로 선택. 라우트는 Port에만 의존.
2. **EXIF 제거**: 업로드 이미지의 EXIF를 서버에서 제거(개인정보 위생) 후 저장.
3. **업로드 엔드포인트**: `POST /api/v1/images`(multipart, 인증 필요) → 저장 후 `{key, url}` 반환. 미인증 401.
4. **DB엔 key만**: 이미지 바이너리는 스토리지, DB엔 key/URL만(WineVintage.representative_image_key 등에서 참조).
5. **R2 어댑터**: Cloudflare R2(S3 호환) 어댑터 구조(자격 env). 실제 업로드는 자격 필요 → 로컬 어댑터로 검증.

## Dev Notes

- **Port 재사용**: `services/ports.py`의 `StoragePort` 이미 정의(Story 1.1). 구현체를 `adapters/`에. [Source: architecture.md#API & Communication / AR4]
- **위생**: 라벨=사물→개인정보 아니나, 사람 우발 촬영·EXIF는 설계로 소거. EXIF 제거는 필수, 사람 감지는 후속. [Source: research OQ-6 / architecture 결정 2-B]
- **R2**: S3 호환 → boto3 endpoint_url=R2. 자격(R2_*) 미설정 시 로컬 어댑터로 폴백(dev). 실제 R2는 계정 자격 필요(범위 밖 검증).
- **의존성 추가**: Pillow(EXIF 제거), boto3(R2). pyproject에 추가.
- **테스트**: 로컬 어댑터+tmp_path. EXIF 있는 작은 JPEG 생성→업로드→저장 파일에 EXIF 없음 확인. 인증은 기존 get_current_user 재사용.
- **프론트 한계**: 실제 카메라 촬영·압축은 기기 필요 → Dart 업로드 서비스(로직)만, 촬영 UI는 입고 흐름(2.6)에서 통합.
