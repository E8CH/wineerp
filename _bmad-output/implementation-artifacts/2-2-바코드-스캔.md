# Story 2.2: 바코드 스캔 (FR3, UX-DR4)

Status: review

## Story

As a 직원,
I want 카메라로 병 바코드를 조준만 하면 자동 인식되기를,
so that 셔터 없이 빠르게 연속 스캔할 수 있다.

## Acceptance Criteria

1. **자동 인식**: mobile_scanner 기반 풀블리드 카메라 + 조준 프레임(ScannerOverlay). 바코드를 대면 셔터 없이 자동 인식.
2. **햅틱 피드백**: 인식 성공 시 햅틱.
3. **인식 값 전달**: 인식된 바코드 값으로 매칭 시도 진입점 호출(매칭 자체는 Story 2.4 — 여기선 콜백/상태로 코드 노출).
4. **실패/재시도**: 카메라 권한 거부·인식 실패 시 명확한 안내 + 재시도(또는 라벨 흐름 전환 자리).
5. **네이티브 설정**: Android 카메라 권한·minSdk, iOS `NSCameraUsageDescription`.
6. **검증**: `flutter analyze` clean, 비카메라 위젯(조준 프레임) 테스트 통과. (라이브 카메라는 실기기 필요 — 범위 밖.)

## Dev Notes

- **패키지**: `mobile_scanner`(ML Kit/CameraX·AVFoundation). 권한은 패키지가 시작 시 요청. [Source: architecture.md 결정 4-B / research]
- **UX**: 풀블리드 카메라 + 골드 코너 조준 프레임, 셔터 없음, 인식=햅틱. 스캔마다 확인 모달 금지(리듬). [Source: ux-design-specification.md#Custom Components ScannerOverlay / Anti-Patterns]
- **매칭 분리**: 인식 코드는 provider/콜백으로 노출만. 실제 code→WineProduct 매칭은 Story 2.4(`crud.wine.find_products_by_barcode`). 중복 연속 인식 디바운스(같은 코드 반복 무시).
- **테스트 한계**: MobileScanner는 플랫폼 카메라 필요 → 위젯 테스트 불가. 조준 프레임(CustomPaint)만 순수 위젯으로 분리해 테스트. 라이브 스캔은 실기기 검증.
- **상태관리**: Riverpod `StateProvider`/`Notifier`로 마지막 코드. go_router 셸 유지.
